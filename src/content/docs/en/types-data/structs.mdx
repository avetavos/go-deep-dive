## Defining a struct

A **struct** groups related fields under a single named type. Each field has a name and a type:

```go
type Point struct {
    X, Y int
}

type User struct {
    Name  string
    Email string
    Age   int
}
```

Fields whose names begin with an uppercase letter are **exported** (visible outside the package). Lowercase names are unexported.

## Struct literals

You can create a struct value in two ways:

```go
// Positional — order must match the field declaration order
p := Point{3, 4}

// Named fields — order does not matter; unset fields get zero values
u := User{Name: "Alice", Email: "alice@example.com"}
```

Named-field form is strongly preferred: it is self-documenting and resilient to field reordering.

## Field access

Use a dot to read or write any field:

```go
fmt.Println(u.Name)   // read
u.Age = 30            // write
```

## Anonymous structs

For one-off shapes — configuration, test fixtures, local grouping — you can define a struct type inline without naming it:

```go
cfg := struct {
    Host string
    Port int
}{Host: "localhost", Port: 8080}
```

## Struct comparison

Two struct values are comparable with `==` if **all** their fields are comparable types. The comparison is field-by-field:

```go
a := Point{1, 2}
b := Point{1, 2}
fmt.Println(a == b) // true
```

Structs containing slices, maps, or functions are **not** comparable with `==`.

## Struct embedding (composition)

Go has no class inheritance. Instead, you **embed** one struct type inside another to promote its fields and methods:

```go
type Address struct {
    City    string
    Country string
}

type Person struct {
    Name    string
    Age     int
    Address          // embedded — City and Country are promoted
}
```

After embedding, `person.City` is shorthand for `person.Address.City`. This is **composition**, not inheritance — the embedded type does not know it is embedded, and the outer type is not a subtype of the inner type.

export const structsCode = `package main

import "fmt"

type Address struct {
\tCity    string
\tCountry string
}

type Person struct {
\tName    string
\tAge     int
\tAddress // embedded struct
}

func main() {
\t// Named-field literal
\tp1 := Person{
\t\tName: "Alice",
\t\tAge:  30,
\t\tAddress: Address{City: "Bangkok", Country: "Thailand"},
\t}
\tfmt.Println(p1.Name, p1.Age)
\tfmt.Println(p1.City)    // promoted from Address
\tfmt.Println(p1.Country) // promoted from Address

\t// Struct copy: p2 is an independent value
\tp2 := p1
\tp2.Name = "Bob"
\tfmt.Println(p1.Name, p2.Name) // Alice Bob

\t// Anonymous struct
\tcfg := struct {
\t\tHost string
\t\tPort int
\t}{Host: "localhost", Port: 8080}
\tfmt.Printf("%s:%d\\n", cfg.Host, cfg.Port)

\t// Struct comparison (all fields comparable)
\ta1 := Address{City: "Bangkok", Country: "Thailand"}
\ta2 := Address{City: "Bangkok", Country: "Thailand"}
\tfmt.Println(a1 == a2) // true
}`;

<Callout title="Structs are value types — assignment copies">
Writing `p2 := p1` produces a completely independent copy of the struct. Every field is copied. Mutating `p2.Name` does not touch `p1`. This is different from slices and maps, where assignment copies only the header and the two variables share the same underlying data. If you need to share a struct across goroutines or avoid copying large structs, use a pointer (`*Person`).
</Callout>
