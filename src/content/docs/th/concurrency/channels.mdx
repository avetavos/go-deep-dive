## Channel คืออะไร?

**Channel** คือท่อส่งข้อมูลแบบ typed ที่ goroutines ใช้สื่อสาร การส่งค่าเข้า channel และรับจาก channel เป็นสองคำสั่งที่ถ่ายโอนทั้งข้อมูลและการ synchronize ในขั้นตอนเดียว

```go
ch := make(chan int)   // unbuffered channel ของ int
ch <- 42              // ส่ง (บล็อกจนกว่า receiver พร้อม)
v  := <-ch            // รับ (บล็อกจนกว่า sender พร้อม)
```

Type system บังคับสิ่งที่ไหลผ่าน channel `chan string` ไม่ยอมรับ `int`

## Unbuffered vs Buffered

| | Unbuffered | Buffered |
|--|-----------|---------|
| **สร้างด้วย** | `make(chan T)` | `make(chan T, capacity)` |
| **Send บล็อก?** | ใช่ — จนกว่า receiver พร้อม | ไม่ — จนกว่า buffer เต็ม |
| **Receive บล็อก?** | ใช่ — จนกว่า sender พร้อม | ไม่ — จนกว่า buffer ว่าง |
| **Synchronize?** | ใช่ — sender และ receiver พบกันในเวลาเดียวกัน | ไม่ — buffer แยกเวลาออกจากกัน |

Unbuffered channels ให้ synchronization guarantee ที่แข็งแกร่งที่สุด: sender รู้ว่า receiver ได้รับค่าแล้วก่อนที่จะดำเนินต่อ

## ทิศทางของ Channel

คุณสามารถจำกัด channel parameter ให้เป็น send-only หรือ receive-only เพื่อแสดง intent ใน type:

```go
func producer(out chan<- int) { out <- 1 }   // send-only
func consumer(in <-chan int)  { v := <-in; _ = v }  // receive-only
```

## การปิด Channel และ Range

Sender เรียก `close(ch)` เพื่อบอกว่าจะไม่มีค่าอีกแล้ว Receivers ใช้ `for range` loop เพื่อดึงค่าทั้งหมดจนกว่าจะ close:

```go
close(ch)
for v := range ch { // exit เมื่อ ch ถูกปิดและว่าง
    fmt.Println(v)
}
```

## Comma-ok Receive Idiom

Receive expression สามารถ return boolean ตัวที่สองที่บอกว่า channel ยังเปิดอยู่หรือไม่:

```go
v, ok := <-ch
// ok == true  → v เป็นค่าจริง
// ok == false → channel ถูกปิดและดึงออกหมดแล้ว v เป็น zero value
```

export const channelsCode = `package main

import "fmt"

func producer(ch chan<- int, n int) {
\tfor i := 1; i <= n; i++ {
\t\tch <- i
\t}
\tclose(ch)
}

func main() {
\t// Unbuffered channel: producer รันใน goroutine
\tch := make(chan int)
\tgo producer(ch, 5)
\tfor v := range ch {
\t\tfmt.Println(v)
\t}

\t// Buffered channel: ส่งทั้งหมดพอดี buffer
\tbuf := make(chan string, 3)
\tbuf <- "a"
\tbuf <- "b"
\tbuf <- "c"
\tclose(buf)
\tfor s := range buf {
\t\tfmt.Println(s)
\t}

\t// Comma-ok: แยกค่าจริงจาก zero value ของ channel ที่ปิด
\tdone := make(chan bool, 1)
\tdone <- true
\tclose(done)
\tv1, ok1 := <-done
\tfmt.Println(v1, ok1)  // true true
\tv2, ok2 := <-done
\tfmt.Println(v2, ok2)  // false false
}`;

<Callout title="ข้อควรระวัง: ส่งเข้า channel ที่ปิดแล้วจะ panic">
การเรียก `close(ch)` แล้วส่งเข้า `ch` จะทำให้เกิด runtime panic การรับจาก channel ที่ปิดแล้วไม่มีทาง panic — มันจะ return zero value ของ type นั้น และ return value ตัวที่สอง comma-ok จะเป็น `false` หลักง่าย ๆ: เฉพาะ **sender** เท่านั้นที่ปิด channel และปิดก็ต่อเมื่อรู้ว่าจะไม่ส่งอีกแล้ว อย่าปิด channel จากฝั่ง receiver
</Callout>
