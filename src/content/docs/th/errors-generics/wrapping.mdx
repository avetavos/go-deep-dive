## ทำไมต้อง Wrap Error?

เมื่อฟังก์ชันเรียกใช้อีกฟังก์ชันแล้วล้มเหลว การคืน error เดิมทำให้เสียบริบท ผู้เรียกเห็นแค่ "database error" แต่ไม่รู้ว่า operation ใดที่ทำให้เกิด request ใดที่เกี่ยวข้อง หรือเกิดที่ layer ไหน การ wrap เพิ่มชั้น context ว่ากำลังทำอะไรอยู่ โดยยังเก็บ error เดิมไว้ให้ตรวจสอบได้

รูปแบบ idiomatic ใน Go:

```go
if err != nil {
    return fmt.Errorf("description: %w", err)
}
```

`%w` คือกุญแจสำคัญ มันทำการ wrap error เดิมไว้ภายใน error ใหม่ รักษา chain ทั้งหมดไว้สำหรับ caller ที่ต้องการตรวจสอบ

## fmt.Errorf และ %w

`fmt.Errorf` สร้าง error ใหม่ที่มี message รวมค่าอื่น แต่วิธีที่มันรวม error argument ขึ้นอยู่กับ verb ที่เลือก:

- **`%v`** จัดรูปแบบ message string ของ error และฝังเป็น plain text type ของ error เดิมหายไป chain ถูกตัด
- **`%w`** บันทึก error เดิมไว้ใน error ใหม่ message ยังมี message เดิม แต่ error เดิมยังสามารถเข้าถึงได้โดยใช้โปรแกรม

```go
base := errors.New("connection refused")

// %v — message only, chain is severed
e1 := fmt.Errorf("dial: %v", base)
fmt.Println(errors.Is(e1, base)) // false

// %w — wraps, chain is preserved
e2 := fmt.Errorf("dial: %w", base)
fmt.Println(errors.Is(e2, base)) // true
```

ใช้ `%w` เมื่อต้องการให้ caller upstream สามารถระบุหรือดึง error เดิมออกมาได้ ใช้ `%v` เฉพาะเมื่อตั้งใจจะทิ้ง error type เดิม เช่น เมื่อ log และ chain ไม่มีความสำคัญ

## errors.Is — ตรวจสอบ identity ผ่าน chain

`errors.Is(err, target)` เดิน unwrap chain ทั้งหมดของ `err` และคืนค่า `true` ถ้า error ใดใน chain ตรงกับ `target` การ match ทำโดย identity (`==`) ตามค่าเริ่มต้น ไม่ใช่ตาม message string

```go
var ErrNotFound = errors.New("not found")

err := fmt.Errorf("getUser: %w", fmt.Errorf("queryDB: %w", ErrNotFound))

fmt.Println(errors.Is(err, ErrNotFound)) // true — found deep in the chain
```

ทำงานได้ไม่ว่า target จะถูก wrap ลึกแค่ไหน นิยาม sentinel error ครั้งเดียว (เป็น `var` ระดับ package) และตรวจสอบได้ทุกที่ใน call stack

## errors.As — ดึง type จาก chain

`errors.As(err, &target)` เดิน chain เช่นกัน แต่แทนที่จะตรวจสอบ identity จะตรวจสอบ type ถ้า error ใดใน chain สามารถ assign ให้ type ของ `target` ได้ จะทำการ assign และคืน `true`

```go
var ve *ValidationError
if errors.As(err, &ve) {
    fmt.Println(ve.Field) // access fields on the concrete type
}
```

ใช้ `errors.As` เมื่อมี custom error type ที่เก็บข้อมูลเพิ่มเติม เช่น field names, HTTP status codes, retry hints และต้องการเข้าถึง field เหล่านั้น ไม่ใช่แค่ตรวจว่า error มีอยู่

## Custom error types

struct ใดก็ตามที่ implement `Error() string` ตรงตาม `error` interface custom types ช่วยให้เก็บ structured context ควบคู่กับ message:

```go
type ValidationError struct {
    Field   string
    Message string
}

func (e *ValidationError) Error() string {
    return fmt.Sprintf("validation: field %q %s", e.Field, e.Message)
}
```

เพื่อให้ custom error type เป็นส่วนหนึ่งของ wrapping chain ให้ implement `Unwrap() error`:

```go
type AppError struct {
    Code int
    Err  error
}

func (e *AppError) Error() string  { return fmt.Sprintf("code %d: %v", e.Code, e.Err) }
func (e *AppError) Unwrap() error  { return e.Err }
```

เมื่อมี `Unwrap` แล้ว `errors.Is` และ `errors.As` สามารถมองผ่าน `AppError` เพื่อไปถึง error ที่มันครอบอยู่

export const wrappingCode = `package main

import (
\t"errors"
\t"fmt"
)

var ErrDatabase = errors.New("database error")

type ValidationError struct {
\tField   string
\tMessage string
}

func (e *ValidationError) Error() string {
\treturn fmt.Sprintf("validation: field %q %s", e.Field, e.Message)
}

func queryDB(id int) (string, error) {
\tif id < 0 {
\t\treturn "", ErrDatabase
\t}
\tif id == 0 {
\t\treturn "", &ValidationError{Field: "id", Message: "must be positive"}
\t}
\treturn "Alice", nil
}

func getUser(id int) (string, error) {
\tname, err := queryDB(id)
\tif err != nil {
\t\treturn "", fmt.Errorf("getUser %d: %w", id, err)
\t}
\treturn name, nil
}

func main() {
\t_, err := getUser(-1)
\tif err != nil {
\t\tfmt.Println(err)
\t\tfmt.Println("is ErrDatabase:", errors.Is(err, ErrDatabase))
\t}

\t_, err = getUser(0)
\tif err != nil {
\t\tfmt.Println(err)
\t\tvar ve *ValidationError
\t\tif errors.As(err, &ve) {
\t\t\tfmt.Printf("field=%q msg=%q\\n", ve.Field, ve.Message)
\t\t}
\t}

\tname, err := getUser(1)
\tif err != nil {
\t\tfmt.Println("error:", err)
\t\treturn
\t}
\tfmt.Println("name:", name)
}`;
