## ภาพรวม Type ใน Go

ทุกค่าใน Go มี type แบบ static ที่กำหนดตั้งแต่ compile time ซึ่ง Go แบ่ง type ออกเป็นสองกลุ่มหลัก:

**Value types** — ตัวแปรเก็บข้อมูลโดยตรง การ assign ค่าจะ copy ข้อมูลทั้งหมด ได้แก่ boolean, ตัวเลข (`int`, `int64`, `float64`, `uint8`, …), และ `string` รวมถึง composite value types อย่าง **array** และ **struct**

**Reference-like types** — ตัวแปรเก็บ header ขนาดเล็กที่ชี้ไปยังโครงสร้างข้อมูลจริงซึ่งถูก allocate ไว้ที่อื่น การ assign จะ copy แค่ header ดังนั้นตัวแปรสองตัวอาจชี้ไปข้อมูลชุดเดียวกันได้ ได้แก่ **slice**, **map**, **channel**, **function**, และ **pointer**

```
┌──────────────────────────────────────────────────────────────────┐
│  ภาพรวม Type ใน Go                                               │
│                                                                  │
│  Value types (copy on assign)    Reference-like (header copied) │
│  ─────────────────────────────   ──────────────────────────────  │
│  bool, int, float64, string      *T  (pointer)                  │
│  [N]T  (array)                   []T (slice)                     │
│  struct { … }                    map[K]V                         │
│                                  chan T, func(…)                 │
└──────────────────────────────────────────────────────────────────┘
```

## เนื้อหาในโมดูลนี้

| บทเรียน | หัวข้อ |
|---------|--------|
| หน้านี้ | ภาพรวม Type และโปรแกรมตัวอย่างแรก |
| Structs | การนิยาม struct, embedding, การเปรียบเทียบ |
| Slices | Array vs slice, `make`, `append`, `copy` |
| Maps | `make(map[K]V)`, comma-ok, delete, การวน loop |

## โปรแกรม composite type แรกของคุณ

โปรแกรมด้านล่างนิยาม struct ชื่อ `Point` สร้าง slice `[]Point` แล้วค้นหาชื่อจาก `map[string]int` ซึ่งแสดงให้เห็น value type (`Point` ถูก copy), slice literal, และการเข้าถึง map พร้อม zero-value เมื่อ key ไม่มีอยู่

export const indexCode = `package main

import "fmt"

type Point struct {
\tX, Y int
}

func main() {
\t// Struct literal with named fields
\tp := Point{X: 3, Y: 4}
\tfmt.Println(p.X, p.Y)

\t// Slice of structs
\tpoints := []Point{
\t\t{X: 0, Y: 0},
\t\t{X: 1, Y: 2},
\t\t{X: 3, Y: 4},
\t}
\tfor _, pt := range points {
\t\tfmt.Printf("(%d, %d)\\n", pt.X, pt.Y)
\t}

\t// Map: string -> int
\tscore := map[string]int{
\t\t"Alice": 95,
\t\t"Bob":   87,
\t}
\tfmt.Println(score["Alice"])
\tfmt.Println(score["Charlie"]) // missing key -> zero value (0)
}`;

<Callout title="Value types ถูก copy; reference-like types แชร์ข้อมูล">
เมื่อคุณ assign struct ไปยังตัวแปรใหม่ — `p2 := p1` — คุณจะได้ copy ที่เป็นอิสระ การแก้ไข `p2` ไม่ส่งผลต่อ `p1` แต่ slice และ map ทำงานต่างออกไป: การ assign จะ copy แค่ header ทำให้ตัวแปรสองตัวชี้ไปข้อมูลชุดเดียวกัน ความแตกต่างนี้คือหัวใจของการตัดสินใจออกแบบส่วนใหญ่ที่คุณจะพบในโมดูลนี้
</Callout>
